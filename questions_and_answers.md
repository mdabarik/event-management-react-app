###### 1. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
let greeting;
greetign = {};
console.log(greetign);
```

- A: `{}`
- B: `ReferenceError: greetign is not defined`
- C: `undefined`

<details><summary><b>Answer</b></summary>
<p>

#### Answer: A

<i>
Output is: {}
In the code greeting and greetign are not the same variable.
Explanation: In JavaScript, when we write greetign = {}, we are creating an object and assigning it to a variable called greetign. In this case, the variable greetign is implicitly declared as a global variable. When we do not use let, var, or const to declare a variable, JavaScript assumes we are creating a global variable. Global variables are accessible from anywhere in our code, which can sometimes lead to unexpected behavior and make our code harder to maintain/understand.


</i>

</p>
</details>

###### 2. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
function sum(a, b) {
  return a + b;
}

sum(1, "2");
```

- A: `NaN`
- B: `TypeError`
- C: `"12"`
- D: `3`

<details><summary><b>Answer</b></summary>
<p>

#### Answer: C

<i>
  We are calling sum function with one integer and one string. javascript converting 1 to string and concatenating '1' and '2'. finally it returned '12'. If we use parseInt on parameter b it will give us, 3. This behaviour is called implicit type conversion.
</i>

</p>
</details>

###### 5. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
let count = 0;
const nums = [0, 1, 2, 3];

nums.forEach((num) => {
  if (num) count += 1;
});

console.log(count);
```

- A: 1
- B: 2
- C: 3
- D: 4

<details><summary><b>Answer</b></summary>
<p>

#### Answer: C

<i>We are iterating the array nums and checking the value. We know 0 is considered as falsy value. That's why for the first iteration where we have element 0, the if statement evaluated false, that's why count is not incremented, so it gives us, 3.</i>

</p>
</details>
